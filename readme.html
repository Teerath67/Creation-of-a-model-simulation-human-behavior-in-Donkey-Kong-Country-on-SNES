BUGS KILLED SO FAR (Since v0.0.6.0):
        32x32 Grid now works trough all 3 ways: keyboard shortcut Ctrl + G, the menu, and the button on the tiles panel. Was a NitroGUI bug.
        Save reporting logic was reversed, causing DKCRE to spit out save failure messages on success and vice-versa.
        Some of the core logic in BFA was screwed up, causing calls to Flush(); to change the last opened file path.
        The ".bak" file saved upon open of a clean ROM was incorrectly using a function that is SUPPOSED to change the filepath. OOPS.
        The Collectibles Panel breaking is fixed.
        AutoLaunch accidentally flushing all in-memory ROM changes is fixed.
        --Since--v0061---------------------------------
        Fully Fixed GFX driver crash: Truncation of Windowed DirectX resolution width to ((width + 3) & ~3), now on both mode swapping as well as init.
        Infinite Browser-Tabs/AutoLaunch bug: Apparently DKCRE will not properly wait for a program to close, so its key state can get stuck while waiting.
        File names were cut off with only the extension in the path. This was due to BFA not setting the path upon opening of existing files. Bug was inconsistent.
        Alt+F4 on system menu (Start Bar Right-Click) is now trapped, since this caused DKCRE to close without warning irrespective of what NitroGUI or DKCRE was doing.
        --Since--v0062---------------------------------
        Fixed NitroGUI's CToolTip class so that Tool Tips do not initialize themselves halfway offscreen when the parent Control is to the far right.
        Eliminated the deadly yet silent IPS-RLE off-by-one bug. Caused viral deletion of tiles and other viral data-corruption mayhem. This was NOT the Coral Capers "bug".
        Continously hitting Page Down to navigate levels could result in an out-of-bounds crash (this was not logged in the v0063 readme).
        --Since--v0063---------------------------------
        If you have a window over DKCRE, and close using the Alt+F4 key, not only do you close the window, you also close DKCRE (the warning box appears as well).
        If you have the Open dialog box opened and press ESC or ALT+F4, you close all of DKCRE (only if ESC shortcut is enabled for the ESC key).
        If you "Save as" then click cancel after the save window pops up, you get an error box that says "No errors". BFA was returning an incorrect status code.
        If you try to "SaveAs..." and are using SMC, and attempt saving to IPS or UPS, DKCRE allows it through without requiring the target ROM.
        Banana Map was not sorting itself after a Nuke operation completed, causing bananas to freeze the game if they were out of order after a Nuke.
        If camera boundaries are on, and you try to Select Level, the arrow's cursor will still be the four-way arrow icon due to focus being ignored.
        In the Tiles Editor (selection mode), you cannot scroll to the left or right automatically. You have to use the Ctrl+Left or Right arrows manually.
        You may see tooltips From the Tiles Panel when it is deactivated. This was a NitroGUI focus bug related to CToolPanel.
        Close the current ROM or IPS, then reopen. The currently selected panel icon will still be on, but the panel itself will be closed (and opposite for the Tiles Panel).
        The Edit menu sticks open if you are in selection mode with the tiles panel open and go to the Extraction Mode or Palette Viewer Mode.
        The camera move/size cursor was being displayed and recognised even if it was off the bottom of the picturebox. Caused users to click the scrollbar inadvertantly.
        --Since--v0064---------------------------------
        Coded DKCRE to fix DKC's "Poison Pond" bug, which was due to DUMMY levels using another level's object map offset.
        Coded DKCRE to fix an object layout overflow bug in DKC. The "Misty Mine Bonus #1" level was overflowing into the DYDDY cheat room object map.
        The 2 DUMMY levels have been removed from being selectable by an INI update, to help prevent buggy behavior.
        The beta "1-1 with 1-3 Object Map" is no longer selectable in Extraction Mode due to the Poison Pond bug fix.
        Added the missing confirmation dialog for replacing files when using "Swap Format and Save To" menu options.
        Handling was missing for most of the File menu in Extraction Mode and for a few options in Palette Editor Mode as well.
        --Since--v0065---------------------------------
        NitroGUI dialogs were crashing due to GetParentToolPanel() trying to access the parent of a dialog (which was apparently not an object that had the GetParentToolPanel() method).
        The camera map was using the wrong drawing coordinate functions for rendering terrain via the RTDraw() function.
        The camera map editing now correctly disables itself upon navigation to a level that is vertical or has no tilemap (DK's Treehouse).
        --Since--v0066---------------------------------
        DKCRE was crashing when trying to "Nuke ALL Levels" due to accessing now-invalid level digits (said digits were buggy in previous builds).
        DKCRE was crashing when trying to Nuke a level that had no Camera Map, due to a division by zero in the "evenly distribute areas" calculation.
        A missing file replace confirmation dialog has been added to the SavePNG and Tiles-Only SavePNG buttons.
        A few technical glitchs were fixed in the ABL and AVG graphics blenders, responsible for the pink when objects are moved/selected outside the level.
        --Since--v0067---------------------------------
        BFA should have been stripping the ROM header when using the SetTargetDataSNES() method. Caused IPS patching troubles.
        Display error with EXtraction sub-panel displayed over other modes was fixed.
        Removed a crash on level $00 during extraction of level tilemaps. Happened due to a digit list error of some sort.
        Extraction checkbox logic bugs removed: If you check off Bonus Entrance Cannons, Show Barrel Meta Cannon Data is disabled. If Bosses was checked, that is unchecked.
        --Since--v0068---------------------------------
        All of the input bugfixes and updates from DELTA. Fixes menu unresponsiveness/lags.
        RE bug: Palette Editor mode was not resetting the dropdown boxes correctly (error after switching to a different mode and back).
        NGUI bug: Menu highlights weren't disappearing when mouse is not over the menu dropdown.
        IPS Creation - "Exploding RLE Segment Count" Bug - happens when certain files are compared to create a patch, such as v1.0 of DKC vs v1.0 of DKC2. Eats a ton of memory very quickly and may lead to system instability. DIAGNOSIS: The size of the vectors in-program are actually the same/ok, but they reserve TONS of excess space, which is why the memory amount explodes when a lot of diffs exist.
        Ctrl+G bug - On startup and after a level is loaded, the shortcut doesn't work until it's pressed again. Caused by a NitroGUI bug whereby the menu controls needed to call Proc() at least once (to initialize) before grid flag could be changed.
        SaveState Bug - The AutoLaunch save state was from the wrong location, according to Mattrizzle.
        NitroGUI Bug - The Hz rate of the Logon Screen can be different from your desktop and having NitroGUI boot up during this time may cause a Hz/speed malfunction in the engine if the Hz differs from the usual value.
        Further NGUI fixes to completely exterminate all input lag. NGUI is now single threaded. Mostly the lag was happening after heavy processing. Bug also affects DELTA.
        Internal UTF-8 Unicode Support. (Language pack features to arrive later!)
        Change the DKCRE info location in the DKC ROM, undo old info from existing hacked roms - When originally designing the editor I mistakenly overwrote part of the SNES interrupt vector pointer table that was filled with junk data to store DKCRE's header information. This will be moved to $3F7F00 in all version of DKC1. The other two games do not currently have a defined location.
        Recalculate internal ROM checksum to avoid showing emulator "checksum errors".

BUGS THAT ARE PROBABLY FIXED, BUT COULD STILL BE HIDING (Since v0.0.6.0):
        The "Coral Capers bug" is due to a missing DKCRE "feature":
                It refers to how editing objects in a vertical level can cause them to all disappear permanently in-game.
                This appears to be due to the camera scripts in vertical levels, which DKCRE cannot interpret.
                The "Coral Capers bug" happens due to objects being incorrectly sorted - but DKCRE cannot possibly know how to sort them without the camera script data.
                This would be a v0.0.7.0 (or later) feature, as the magnitude of the fix is too large to just be for a regular "bug fix" update.

        No other actual DKCRE bugs have been found, but users have reported some issues with DKC hacking:
                You may experience black screens or seemingly random freezes based on Banana layouts. This is a fatal bug within DKC itself! Cause Unknown.
                Objects may dissapear "permanently" in-game, if certain undetermined objects/sequences-of-objects are placed in your level.

Editor Controls:
Ctrl + OOpen an Image or Patch+Image combination.
Ctrl + SSave to Current Format
Ctrl + Shift + SSave As (Accepts .SMC, .SFC, .IPS, and .UPS)
Alt + MOpen Music/Sound Settings Dialog
Ctrl + Shift + Alt + DeleteNuke Current Level
Ctrl + GShow/Hide 32x32 Grid
F4Game Autolaunch
F5Level Autolaunch
F8Refresh GameConfig INI Data From On-Disk Copy
F11Toggle Fullscreen Mode (Also Alt+Enter)
Alt+F4 (or ESC if enabled)Exit Program (CONFIRMATION DIALOG)
Ctrl + Arrow KeysScrolls in the desired direction, even with something selected.
HomeGo to Start of the current Level
EndGo to End of the current Level
Home + EndGo to Middle of the current Level
Page UpGo to Next Level in the Levels List
Page DownGo to Previous Level in the Levels List
Ctrl + Page UpGo to Parent Level (only if applicable)
Double Right-Click an ObjectGo to Destination Bonus Level (only if object is bonus entrance or barrel)
Alt + Shift + B + 1Changes displayed bananas to Normal Bananas. (These shortcuts DO NOT modify the game's actual bananas!)
Alt + Shift + B + 2Changes displayed bananas to Winky Bananas.
Alt + Shift + B + 3Changes displayed bananas to Expresso Bananas.
Alt + Shift + B + 4Changes displayed bananas to Rambi Bananas.
Alt + Shift + B + 5Changes displayed bananas to Enguarde Bananas.
                                                   

Tile Editing and the Tiles Panel (Editing Mode):
TABSwitch Between Selection and Editing Mode
Left-ClickChanges tiles on the front layer (blue).
Right-ClickChanges tiles on the back layer (red).    Useful only in Mine Cart Carnage.
Ctrl + Left-ClickSelects in the Tiles Panel the front layer (blue) tile you just clicked.
Ctrl + Right-ClickSelects in the Tiles Panel the back layer (red) tile you just clicked.    Useful only in Mine Cart Carnage.
                                                   

Tile Editing and the Tiles Panel (Selection Mode):
TABSwitch Between Selection and Editing Mode
Left-Click and DragSelects tiles on the front layer (blue).
Right-Click and DragSelects tiles on the back layer (red).    Useful only in Mine Cart Carnage.
Ctrl + Left-Click and DragSelects MORE tiles on the front layer (blue).
Ctrl + Right-Click and DragSelects MORE tiles on the back layer (red).    Useful only in Mine Cart Carnage.
1Moves your selection to the front layer (blue) if it was on the back layer (red).
2Moves your selection to the back layer (red) if it was on the front layer (blue).
Arrow KeysMove Selection by 1 Tile.
Ctrl + ASelects the entire level (to select entire back layer, select some tiles on the back layer before pressing this).
Ctrl + XCut selected tiles to the Windows Clipboard.
Ctrl + CCopy tiles to the Windows Clipboard.
Ctrl + VPaste tiles from the Windows Clipboard as a selection. Paste location is determined by your mouse pointer.
PLEASE NOTE: You may only cut/copy/paste between levels of the same acrhetype.
Ctrl + FFlip entire selection of tiles by X Axis (horizontally).
Ctrl + Shift + FFlip entire selection of tiles by Y Axis (vertically).
DeleteDeletes the currently selected tiles.
                                                   

After selecting an object:
Click & DragMove Object
Arrow KeysMove Object by 1 Pixel
Hold Down G KeySnaps the object to an 8x8 grid as you move
Delete KeyDelete Object (Becomes $A51F Banana Bunch by default)
Ctrl + Page DownGo to Destination Bonus Level (only if object is bonus entrance or barrel)
The following use Hexadecimal numbers (**You do not need to hit enter**):
(key sequence) X####Sets Object X Position
(key sequence) Y####Sets Object Y Position
(key sequence) C####,####Sets Object Position (typing the comma is optional)
(key sequence) D####Sets Object Digit
(key sequence) L####Lookup an Object in the Panels by its Digit (also works without selected object)
Space KeyLookup a selected Object in the Panels
                                                   

While editing Camera Boundaries:
Click & DragMove Focus Area (Grab borders to move only that border)
Right-ClickChange Reaction Speed of Focus Area. Blue/Teal/Green means slow, Red/Orange/Yellow means fast.
